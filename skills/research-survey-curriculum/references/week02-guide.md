# Week 02 — 지식 창고 + 검수 기준

> 스터디 정본 2주차("지식 창고 + 검수 기준 — 리뷰 패널·검증 게이트·스킬 분해")를 매핑.
> 목표: 반입한 자료를 **2분할 위키 노트**로 정본화하고, **결정론 검증 게이트**와 **채점
> 기준(gold 질문셋·오류 매설)**을 직접 만들어 "검수가 되는지"를 수치로 확인한다.

## 이번 주 목표
- 검증한 내용을 재검색 가능한 정본(위키 노트)으로 쌓는다 — 2분할 스키마(Compiled Truth/Timeline).
- 승격 게이트가 "출처 없음·이력 변조·필수키 누락"을 기계적으로 막는 걸 직접 본다.
- 요약이 원문에 실재하는지 대조하는 source-coverage 검수를 켠다.
- gold 질문셋과 오류 매설 노트로 **검색·검수 품질을 채점**한다(감→수치).

## 지금 할 일

### 1. 2분할 위키 노트 이해 + 승격 게이트
- 노트 스키마(정본은 워크스페이스 `00-system/data-dictionary.md`): frontmatter 필수키
  `id/title/created/tags` + 출처(source 또는 본문 페이지/표/arXiv/DOI 인용) + 본문 2분할 —
  `## Compiled Truth`(항상 현재값·새 증거 시 통째 REWRITE) + `## Timeline`(append-only·기존
  항목 수정 금지).
- 검증 통과 산출물만 정본으로 **승격**한다. 게이트는 기본 dry-run(diff 미리보기) → 승인 →
  apply. 게이트가 막는 것(직접 봐 두기):
  - 출처 인용 0 → 거부(B8) · 기존 Timeline 항목 변조 → 거부(E1 append-only) ·
    필수키 누락 → 거부(A3) · workspace 밖 파일 → 거부(위치 게이트).
- 이 게이트는 research-survey-run 스킬이 `wiki_promote`로 실행한다(정본 직접 쓰기 금지).

### 2. source-coverage 검수 (누락·오출처를 결정론으로)
- 요약의 **Evidence 절 수치·직접인용(12자+ 따옴표)**이 원문(초록/PDF 추출 텍스트)에 실제로
  있는지 grep 대조한다 — 없으면 FAIL(발명 수치·오인용 차단). 초록 키포인트 커버율이 임계
  (기본 0.6) 미달이면 경고(누락 신호). `verify_summaries --corpus`가 켠다.
- ★한계 정직: 이 대조는 **수치·직접인용만** 본다. Summary의 질적 허위·의미 왜곡은 이 게이트
  범위 밖이다(week04의 교차 벤더 리뷰가 그 사각을 보완).

### 3. 검수 기준 만들기 — gold 질문셋 + 오류 매설
- **gold 질문셋**(`00-system/wiki-gold.json`, 샘플 `wiki-gold.sample.json`): 질문 + 기대 1위
  노트 id + 기대 근거 문구. 각 근거 문구는 노트 본문에 글자 그대로 있어야 한다(채점기가
  fail-closed로 검사 — 기준 자체의 오염 차단).
- **오류 매설 노트(ev)**(`40-drafts/ev/`, 샘플 5종): 발명 수치·문구 오인용·출처 없음·
  Timeline 변조·필수키 누락 + 정상 대조군 1개. 검수 게이트가 **거부해야 정상**인 노트들
  (대조군은 통과해야 정상 — 과차단 감시).

### 4. 채점 (감 → 수치)
- gold로 **1위 적중률·top-k recall**, 매설로 **거부율**을 산출한다. 동봉 샘플 실측치는
  적중 12/12·매설 거부 5/5·대조군 통과다. 채점 리포트는 `70-analysis/wiki-grade-report.json`.
- 채점은 research-survey-run 스킬이 `wiki_grade`로 실행한다. 판정은 전부 결정론 채점기 —
  LLM 자평을 점수 근거로 쓰지 않는다.

## 완료 체크리스트
- [ ] 내 논문 1편으로 2분할 위키 노트(필수키·출처·CT/Timeline)를 만들어 승격했다(dry-run→apply).
- [ ] 일부러 틀린 산출물(출처 없음·수치 발명)이 게이트에서 거부되는 걸 확인했다.
- [ ] `verify_summaries --corpus`로 요약의 Evidence 인용 실재·키포인트 커버율을 대조했다.
- [ ] gold 질문 5개 이상 + 오류 매설 노트를 내 주제로 만들었다.
- [ ] 채점을 돌려 **1위 적중률·거부율 수치**를 리포트로 받았다.
- [ ] 발표 한 줄: "제 직무는 ___, 검수 대상은 ___, 전에는 눈으로 봤고 이번엔 하네스로 자동 대조로 바꿨어요."
