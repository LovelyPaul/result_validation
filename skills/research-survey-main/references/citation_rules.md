# citation_rules — 출처·근거·인용 규칙

> 서베이의 신뢰성은 "모든 수치·주장이 원문으로 되짚어진다"에서 나온다. 이 규칙은 요약·인사이트·
> 서베이·가설 전 산출물에 적용된다.

## 요약 Evidence 규칙
- Summary/Evidence의 **모든 수치·비교 주장**에 원문 위치를 붙인다: `— Table 3, p.6` / `— §4.2, p.8` / `— Figure 2`.
- 카테고리당 페이지 인용 **최소 3개**(자가검증 기준). 인용 없는 서술은 승격 불가.
- **초록 복붙 금지** — PDF 본문(방법·실험·한계 절)을 실측한 근거여야 한다.
- 각 요약 끝에 `source_pdf: <파일명> (p.1 제목 대조 OK)` — 엉뚱한 논문 요약 방지.

## 삼각검증·출처 등급 (연구 워크스페이스 표준)
- **삼각검증**: 핵심 주장은 최소 2개 독립 출처로 교차 확인. 단일 출처 주장은 그 사실을 명기.
- **출처 품질 등급**: A(공식/원문·1차) · B(피어리뷰 2차) · C(프리프린트/공식블로그) · D(보도/요약) ·
  E(개인/불명) · F(생성물·미검증). 등급을 인용에 병기하고, 낮은 등급 단독 근거는 승격 보류.
- **데이터 규칙**: 출처 없는 데이터 금지 · 원본 직접 수정 금지 · 검증 없는 인용 금지.

## 버전·시점 규칙
- 라이브러리·모델·수치는 **버전/시점 명시**(버전 없는 API 서술은 함정 생산기). 학회 논문은 venue·연도.
- camera-ready 개제는 원제목 유지 + 개제 note 병기(초록 축자 일치 + 동일 forum/id = 동일 논문).

## 인사이트·가설의 근거
- 인사이트의 클러스터·트렌드·갭은 **초록에서 확인된 것만**. 근거 논문 제목을 병기. 과장·일반론 금지.
- 가설은 falsifiable 규격 + **근거 논문** + **예상 반론** 1줄씩. 근거가 딥다이브(전문 정독)면 정독 사실 명기.

## 승격 게이트에서의 출처 검증
- 서베이 정본 승격 시 cold-read 좌석이 ①인용 경로가 실제 존재하는가 ②본문 주장을 지지하는가를
  대조. 발명 인용·근거 없는 단정 발견 = 반려.
