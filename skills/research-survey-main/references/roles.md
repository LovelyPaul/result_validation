# roles — 다중 노드 협업 시 역할과 통신

> 단독 실행이면 진행자 에이전트 1개가 전 단계를 수행하되 **검증만은 별도 서브에이전트/세션으로
> 분리**해 producer≠evaluator를 지킨다. 대규모·지속 서베이에서는 아래 역할을 분리한 팀으로 굴린다.
> (jarvis/cys 환경의 DevTeam 부서가 이 구성의 레퍼런스 구현.)

## 역할 표

| 역할 | 렌즈 | 한다 | 안 한다 |
|---|---|---|---|
| **master (지휘)** | 게이트키퍼(회귀 제로·자료구조 우선) | 요구 분해·위임·독립 검증·심판·게이트 판정·보고 | 요약/구현(위임) |
| **worker** | 단순성·작은 diff | 위임받은 요약/구현. 논문 1편당 서브에이전트 병렬. 자가검증 | 자기 산출 최종 채점 |
| **reviewer** | rigor·cold-read | fact-verify(수치·인용·과장)·PDF 표본 대조·가설 novelty 채점 | 산출물 생성 |
| **inspector** | 중립·누수 감사 | 보안/누출·의존성·라이선스 + 가설 검증설계 자기참조/누수 감사 | (페르소나 없음, 중립 유지) |
| **cso** | 시스템 거버넌스 | 자원 감시·노드 회생·인프라 수리 | 콘텐츠 생성 |

## 작업 순서 (누가 언제)
```
master   : 다이얼 조정 → 분류 실행 → 스켈레톤 생성 → [요약 위임] →
           (대기) → 독립 실측 검증 → [표본 대조 발주] → (상충 시 원문 실측) →
           정리 업로드 → 인사이트 발주 → 서베이 승격 승인 → 가설 채점 발주 → 판정 → 보고
worker   : 위임 수령 → 병렬 PDF 실측 요약 → 자가검증 → 완료 보고 (의심 시 플래그·판정 대기)
reviewer : 서베이 cold-read · 요약 PDF 표본 대조 · 가설 채점 → 결함/PASS 보고 (못 읽으면 blocking)
inspector: 가설 검증설계 누수 감사 · 보안/누출 점검
cso      : 상시 거버넌스 · 인프라 수리(중대 조치는 master 인가 후·진단은 읽기전용)
사용자   : 관심 주제 지정 · 다이얼 조정 방향 · 서베이 승격 승인 · 외부 발행/비가역 결정 · kill-switch
```

## 통신 규약 (jarvis/cys 기준 — 다른 오케스트레이터도 개념 동일)
- **양방향 push**: 노드가 서로 stdin에 메시지를 쏜다(폴링 아님). `send` + `send-key Return`.
- **위임 티켓**: 수기 위임 금지 — 생존 확인 + 절대강조 4규칙을 자동 주입하는 도구로 발권.
- **리뷰 의뢰**: 제약·cold-read를 자동 주입하는 도구로. 저자 의도는 손으로도 넣지 않는다.
- **승인**: 정상 작업(run·update)은 즉시 allow. 외부 발행·비가역은 사용자에게 격상.
- **컨텍스트 사이클**: 노드가 컨텍스트 % 자기보고 → 임계에서 저장→검증→clear→재주입. 저장 없이 clear 금지.

## 폴백 규약
- 특정 리뷰어(벤더) 쿼터 소진/부재 시 다른 좌석이 겸임 — 벤더 다양성 약화는 보고에 정직 라벨링.
- 단독 환경: 검증 좌석을 최소한 "새 세션/서브에이전트"로 분리해 cold-read를 흉내(같은 컨텍스트 자가채점 금지).
