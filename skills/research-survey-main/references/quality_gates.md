# quality_gates — 심의와 검증 규칙

## 5대 원칙
1. **증거 기반 완료** — "됐다"는 말이 아니라 실행된 출력·diff·PDF 인용만이 완료. 완료 보고를
   그대로 믿지 않고 직접 확인 후 승인.
2. **producer ≠ evaluator** — 만든 좌석은 자기 산출물을 채점하지 않는다. 검수는 cold-read
   (저자 의도·세션 컨텍스트 미주입)로 독립 좌석이 수행.
3. **환각 0** — 근거 없는 단정 금지. 출처(PDF 페이지·커밋·명령 출력)만이 사실. 모르면 물러난다(precision 우선).
4. **축적** — 살아남는 아티팩트 없는 작업 = 설계 결함. 요약·인사이트·서베이·가설·판정을 durable히 남긴다.
5. **결정론 환원** — 카운트·게이트 판정·진행률은 LLM 추론이 아니라 스크립트 출력으로만 확정.

## 3중 검증 (6단계 게이트)
1. **자가검증(결정론)**: 만든 좌석이 grep으로 플레이스홀더 0·섹션 완비·페이지인용 ≥3·source_pdf 확인.
   — 1차일 뿐(producer이므로 최종 아님).
2. **독립 실측**: 같은 결정론 검사를 만든이가 아닌 좌석(진행자/master)이 재실행해 보고와 대조. 눈대중 금지.
3. **표본 원문 대조(cold-read)**: 카테고리당 2편을 독립 검수 좌석이 PDF와 대조 — 발명 수치·비교
   방향 왜곡·페이지 오인용 적발.
- **상충 시**: 리뷰어 점수가 이상치이거나 좌석 간 판정이 엇갈리면 **원문(PDF) 직접 실측이 최종심.**
  리뷰어 판정도 맹신하지 않는다.

## 콘텐츠 무결성 규칙 (실측이 만든 것)
- **source_pdf 대조**: 요약 착수 전 PDF p.1 제목 ↔ 목록 제목 일치 확인 + 기록. 불일치 = 요약 금지·플래그.
- **개제(改題) 판정**: 초록 축자 일치 + 동일 forum/id = 동일 논문(camera-ready 개제) → note 후 진행.
- **초록↔본문 수치 불일치**: 본문(표)을 정본 채택하고 불일치를 명기(논문 자체의 초록-결론 차이 대비).
- **파일 미열람 판정 무효**: 검수 좌석이 원문을 못 읽었으면(도구 오류 등) 점수 대신 "검증 불가=blocking"
  신고 — 못 본 채 점수 매기는 것 금지.

## cold-read 계약
- 검수 좌석에 주는 것: ①산출물(요약/서베이/가설) ②공개 스펙(수치·인용 규칙)뿐.
- **금지**: 저자 의도·세션 히스토리·"이렇게 하려 했다" 서술 주입. 의뢰문은 제약을 자동 주입하는 도구로 생성.

## 가설 accept_threshold
- 예: `feasibility >= 4 && (novelty + impact) >= 6`. **자기채점 금지** — 외부 2좌석(reviewer·inspector)이 매김.
- 미달 = **hold**(실패 아님 — 게이트가 작동한 것). 사유·개정 트리거를 ledger에 영속.
- 반복 재개정(점수만 올리려는 gaming) 금지. hold를 살리려면 근거 논문 전문 정독 딥다이브로 갭을 재정의.

## 절대 강조 4규칙 (위임 시 자동 주입)
- a) 품질 절대우선(속도·토큰은 이유 못 됨) b) 할루시네이션 방지(근거 필수) c) 의도 합의(모호하면
  합의까지 질문) d) 요약·압축 금지(수치·표·단서 보존, 용어만 쉽게 풀되 길이 유지).

## 출처 등급 (연구 워크스페이스 표준 연계)
- 삼각검증(최소 2 출처) + 출처 품질 등급(A~E/F). "출처 없는 데이터 금지·원본 직접 수정 금지·
  검증 없는 인용 금지." 상세: citation_rules.md.
