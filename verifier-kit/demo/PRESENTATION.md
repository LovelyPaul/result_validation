> 하나는 '근거가 추적되는 요약'입니다. 맨몸은 'SOTA를 달성했다'처럼 그럴듯하지만 출처가

> **문제는 — 이 둘을 사람이 눈으로 구분할 수 없다는 겁니다.** 그래서 오늘, 이걸 사람이 아니라
> 코드가 구분하게 만드는 하네스를 뜯어보고, 제 것으로 지은 과정을 보여드리겠습니다."

아티팩트 1을 스크롤하며 3가지만 짚는다 (전체 낭독 금지):

1. **§01 한 줄 명제** — "이 패키지가 파는 건 논문 서베이 기능이 아니라, LLM 산출물을

   "LLM 채점이 아니라 정규식과 diff가 판정한다"를 강조.
3. **§06 게이트를 채점하는 게이트** — "검수기를 어떻게 믿나? 오류를 일부러 심어서 거부율을

> 멘트: "해부하고 나니, 이걸 제 도메인 검수 에이전트로 넓히려면 뭘 보완해야 하는지 6가지가

### 막 2 — 구축 (5분) · **아티팩트 2**

아티팩트 2의 KPI 밴드부터 (3 에이전트 · 100% · 4/4 self-test · 6/6 보완점).

1. **§02 구조** — 3층 + 다이얼 그림. **권한 경계**를 손가락으로 짚는다:
   "초록(verify/grade)만 거부 권한. 노란(semantic)은 WARN만. 이 선이 하네스의 척추."
2. **§03 여정** — 6대 보완점이 커밋으로 쌓인 타임라인. 빠르게 훑고.
3. **§04 재사용성** — 세 에이전트 카드(논문·코드·문서). "전부 같은 코어 무수정으로 붙었다."
   → **여기서 결정적 교훈**: "코드 리뷰에서 PR#를 근거로 오인한 버그가 있었는데,
   **코어가 아니라 다이얼에서** 고쳐졌습니다. 도메인 문제가 도메인 파일에서 풀린다 —
   설계가 옳다는 신호였죠."

**전환**: "말로만 하면 안 믿기시죠. 라이브로 보여드리겠습니다." → 터미널.

### 막 3 — 라이브 데모 (6분) · **터미널** ★하이라이트

**데모 A — 재사용성 (1분)**
```
cat dials/code-review.json        # 다이얼이 도메인을 정한다 (코드 아님)
bash ci/run-gate.sh               # 0.25초 — 세 에이전트 다 채점, 전부 100%
```
> "세 도메인이 하나의 엔진으로. 새 에이전트는 코드가 아니라 JSON 한 장."

**데모 B — 발명 수치 잡기 (1분)**
```
# 매설 노트 하나를 열어 보여주고 (발명 수치 94.2% / 92.4)
python3 verify_core.py --dir ev/research-paper --dial dials/research-paper.json \
  --corpus ev/research-paper/corpus.sample.json
# → [FAIL] ... 근거 수치 원문 부재: '92.4'
```
> "LLM에게 '이거 맞아?' 안 물어봅니다. 원문에 그 숫자가 있나 grep 합니다. 결정론."

**데모 C — 게이트 깨뜨리기 (3분) ★클라이맥스**
```
bash demo-break-the-gate.sh       # 4단계 자동 (⏎로 진행)
```
- 1) 지금 통과 → 2) 매설을 정상으로 바꿔 검수기 약화 → 3) **거부율 100%→75%, exit 1 차단**
  → 4) 원복
> "누가 검수기를 약하게 만들면 — 다이얼을 느슨하게 풀든, 매설을 우회하든 — CI가 자동으로
> 막습니다. **검수 하네스가 스스로를 감시합니다.**"

**전환**: GitHub 탭으로. "이게 제 노트북만이 아니라 —"

## Q&A 대비 (예상 질문 + 답)

| 질문 | 답 |
|---|---|
| "grep으로 다 잡히나? 의역·왜곡은?" | 못 잡는다 — 그래서 의미 채널을 WARN으로 얹었다. 하지만 그것도 한계(의역~0.39 구분 불가)를 실측하고 그 안에서 설계. 정밀 판별은 임베딩/NLI 로드맵. **정직하게 한계를 말하는 게 포인트.** |
| "거부율 100%면 완벽한 거 아닌가?" | 아니다. 매설 스위트가 잡도록 만든 오류를 잡을 뿐. 실전 신뢰도는 매설을 *내 도메인 실제 실패 모드*에서 계속 늘려야 오른다. |
| "왜 LLM에게 채점 안 시키나?" | 비결정론이라 재현·감사 불가. 같은 입력에 다른 판정이 나오면 게이트가 아니다. LLM은 생성만. |
| "새 도메인 붙이는 데 얼마나?" | 다이얼 JSON 1장 + 매설 스위트. 코어·CI 무수정. 문서 팩트체크는 첫 판에 100%. |
| "이관하면서 원본 안 건드렸나?" | 코어는 무수정(git diff 0). 단 이관 중 원본에도 있던 버그 2개(CJK 소수·지표명 숫자)를 발견해 고치고 회귀 테스트로 박았다. |
